# 이더넷

> 통신 매체를 통해 신호를 송수신하는 방법

데이터 링크 계층에서 주고받는 데이터 형식이 정의된 기술

대부분의 LAN은 이더넷을 기반으로 구현
> 

## 이더넷 표준

이더넷은 IEEE 802.3이라는 이름으로 국제 표준화된 기술 → 네트워크 장비 제조사가 달라도 호환됨

![image.png](attachment:bdcaf88a-d7d3-4131-b2cf-e70c9c82cf35:image.png)

- LAN 대부분이 이더넷 표준을 따름 → 대다수의 LAN 장비들이 특정 이더넷 표준을 따름
- 이더넷 표준이 달라지면 통신 매체의 종류를 비롯한 신호 송수신 방법, 최대 지원속도도 달라짐

## 이더넷 프레임

> 이더넷 기반의 네트워크에서 주고받는 프레임
> 
- 오늘날 LAN은 대부분 이더넷을 기반으로 구현 → 프레임이 이더넷 프레임을 지칭한다 해도 무방

![image.png](attachment:4aaaafea-b56c-4a47-9327-64ed80c924dc:image.png)

**✅ 프리앰블**

프레임의 시작을 알리는 준비 신호

- 첫 7바이트 : 10101010
- 마지막 바이트 : 10101011

**✅ 송수신지 MAC 주소**

네트워크 장치마다 부여된 고유한 하드웨어 주소

- 내 노트북의 랜카드, 공유기 포트, 서버의 네트워크 카드가 가지고 있는 주소

```java
00:1A:2B:3C:4D:5E
```

- 12자리 16진수
- LAN 안에는 여러 장치가 연결되어 있음 → 프레임이 도착했을 때 어떤 장치가 받아야 하는지 구분
- 스위치
    - 목적지 MAC을 보고 어느 포트로 보내야 하는지 결정
    - 자기 MAC 주소와 비교해서 일치하면 받음
- NIC라는 네트워크 장치가 네트워크 인터페이스를 담당 → NIC가 여러 개면 한 호스트가 여러 개의 MAC 주소를 가질 수 있음

**✅ 타입/길이**

- 타입
    - 이 프레임의 데이터가 무슨 종류의 프로토콜 데이터인지 알려줌
    - IPv4인지, IPv6인지, ARP인지
- 길이

**✅ 데이터**

실제로 전달하려는 내용이 존재하는 곳

- 최대 크기는 보통 1500바이트로 제한
- 상위 계층으로 전달하거나 전달받을 데이터
- 네트워크 계층 패킷의 최대 크기를 지칭하는데 사용

**✅ FCS**

프레임의 오류가 있는지 여부를 확인하기 위한 필드

네트워크 선을 통해 데이터가 이동할 때 노이즈, 전기적 간섭, 장비 문제 등으로 비트가 바뀔 수 있음

- CRC라는 값을 만들어 계산 결과가 같으면 정상, 다르면 오류 발생
- CRC는 오류를 검출하는 역할이지 고쳐주는 역할이 아님
- 과정
    - 송신지에서 데이터 +CRC 값을 계산해서 보냄
    - 수신지에서 전달받은 데이터 + CRC 값을 계산
    - CRC 값 대조 → 같으면 오류가 없음

## 전파와 WiFi

![image.png](attachment:0cdfd796-9f58-494b-b99c-8b8ac5d4f676:image.png)

**✅** 전파

- 3kHz~3THz 사이의 진동수를 갖는 전자기파
- 2.4GHz, 5GHz 정도는 알아두자

**✅** 와이파이

![image.png](attachment:21117288-0748-40ce-9a27-3c36f237887d:image.png)

- IEEE 802.11이라는 표준을 따르는 무선 기술
- 표준 규격에 따라 지원되는 최대 속도, 주파수 대역 등이 달라짐
- 와이파이 뒤에 붙는 숫자로 세대를 구분

![image.png](attachment:8f24ef82-18ba-479e-b8e8-13edd8260acb:image.png)

- 같은 주파수 대역을 사용할 수 있다
    - 채널이라는 하위 주파수 대역으로 세분화
    - 채널의 주파수가 서로 중첩되지 않으면 신호 간섭으로 인한 성능 저하가 없음
    - 신호가 중첩되지 않는 채널을 쓰는게 중요

# 네트워크 인터페이스 : NIC

네트워크 상에서 노드와 통신매체가 연결되는 지점

![image.png](attachment:b039c56f-4b35-4666-be7b-6c8a5538d699:image.png)

- 네트워크 인터페이스마다 MAC 주소가 부여
- NIC라는 하드웨어가 역할을 함
- 통신매체의 신호 → 호스트가 이해하는 프레임으로 변환, 호스트가 이해하는 프레임을 통신 매체의 신호로 변환
- 동작
    - NIC를 작동시키는 시스템 콜이 호출
    - 송수신이 수행되고, 입출력이 완료되면 인터럽트를 통해 CPU에게 작업이 완료되었음을 알림

# 허브와 스위치

물리 계층과 데이터 링크 계층의 중간 노드

## 물리 계층의 허브

![image.png](attachment:89367d3b-91e0-4312-9be8-4f209eddd62e:image.png)

- 여러 대의 호스트를 연결
- 케이블 커넥터가 꽂히는 부분, 통신 매체를 연결하는 지점은 포트라고 함
- 특징
    - 전달받은 신호를 모든 포트로 보냄
    - 반이중 모드로 통신
        - 송신 또는 수신을 번갈아 가면서 수행하는 방식
        - 한 쪽이 송신하면 다른쪽은 송신이 불가능

## 데이터 링크 계층의 스위치

![image.png](attachment:8f3a6007-a187-4971-91bd-fec0a4d42f63:image.png)

허브의 한계를 보완하기 위한 네트워크 장비

- 전달받은 신호를 목적지 호스트가 연결된 포트로만 보냄
    - 스위치에 MAC 주소 학습 기능이 있음
    - MAC 주소 테이블
        - 포트, 연결된 호스트의 MAC 주소의 대응 관계를 메모리에 저장
        - 이를 참고해서 전송
- 전이중 모드를 지원

![image.png](attachment:44243c74-9f89-4300-86e4-ad158ec7f38c:image.png)

- VLAN
    - 같은 스위치에 연결된 호스트를 하나의 네트워크로 간주하고 싶지 않을 때 → 논리적인 네트워크로 나눔
    - 서로 통신하려면 네트워크 계층 이상의 장비가 필요
