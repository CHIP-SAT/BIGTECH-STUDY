## 이더넷

- 통신 매체를 통해 신호를 송수신하는 방법
- 데이터 링크 계층에서 주고받는 데이터 형식 등이 정의된 기술

**이더넷 표준**

- 오늘날의 유선 LAN 대부분이 이더넷 표준을 따르기 때문에 대다수의 LAN 장비들이 특정 이더넷 표준을 따름
- 이더넷 표준이 달라지면 통신 매체의 종류를 비롯한 신호 송수신 방법, 최대 지원 속도도 달라질 수 있음.
    - 최신 이더넷 표준을 준수하는 네트워크 장비가 일반적으로 최대 지원 속도가 빠름.

## 프레임

- 프레임은 `프리앰블`, `수신지`, `MAC 주소` 등의 정보를 포함하고 있음.

![image.png](attachment:61d6ce29-4af5-4b9e-8bb4-f9ec5b0fbf4f:image.png)

**프리앰블**

- 송수신지 동기화를 위해 사용되는 8바이트 크기의 정보
- 수신지는 이 프리앰블 비트를 통해 불규칙한 전기 신호 속에서 의미 있는 데이터의 시작점을 찾을 수 있음.

**송수신지 MAC 주소**

- 프레임에는 송신자와 수신자를 특정할 수 있는 6바이트 길이의 MAC 주소가 명시됨.
    - MAC 주소: 네트워크 인터페이스(= 네트워크를 향하는 통로)마다 하나씩 부여되는 주소
- 보통 NIC라는 장치가 네트워크 인터페이스를 담당
    - NIC: 디지털 신호 ↔ 전기 신호/광 신호

**타입/길이**

- 타입: 캡슐화된 상위 계층의 정보를 의미
    - 따라서 타입을 통해 어떤 상위 계층 프로토콜이 캡슐화되었는지 알 수 있음.

**데이터**

- 데이터 필드에 포함될 수 있는 데이터의 크기는 일반적으로 1500바이트 이하로 제한돼 있음.
    - 1500바이트가 넘어가면 → 데이터가 여러 패킷으로 나뉘어 보내짐.

**FCS**

- 프레임의 오류가 있는지 여부를 확인하기 위한 필드
- CRC라는 오류 검출용 값이 명시됨.
- 송신지 → [`데이터` + `CRC`] → 수신지
    - if (`송신지에서 보낸 CRC 값` == `수신지에서 데이터에대해 계산한 CRC 값`) then 오류 X

## 유무선 통신 매체

**유선 매체 - 트위스티드 페어 케이블**

- 구리선을 통해 전기적인 신호를 주고받기 때문에 전기 신호에 왜곡을 줄 수 있는 주변 잡음에 취약함.

**무선 매체 - 전파와 Wifi**

- 진동수 2.4GHz와 5GHz는 와이파이를 사용할 때 주로 활용됨.
- 별개의 무선 네트워크는 같은 주파수 대역을 사용하더라도 서로의 신호에 간섭하지 않아야 함. → **채널** 사용
    - 채널 대역에서  무선 통신이 이루어짐.

## NIC

> 네트워크 상에서 노드와 통신 매체가 연결되는 지점
> 
- 통신 매체의 신호 ↔ 프레임
- NIC의 지원 속도는 저마다 다르고, 이는 네트워크의 속도에 큰 영향을 끼침.
    - 아무리 외부에서 `1G`의 데이터를 쏴줘도, 내 컴퓨터의 대문(NIC)이 초당 `100M`만 받아들일 수 있다면 나머지 `900M`는 들어오지 못하고 대기

## 허브

> 물리 계층의 대표적인 네트워크 장비로, 여러 대의 호스트를 연결
> 
- **허브의 중요한 2가지 특징**
    - 전달받은 신호를 모든 포트로 내보냄.
    - 반이중 모드로 통신
        - 반이중 모드: 동시 송수신이 불가능한 상태로, 송신 또는 수신을 번갈아가며 수행
        - 전이중 모드: 동시 송수신이 가능한 상태(스위치)
- 허브에서 충돌이 발생할 수 있는 영역(= 콜리전 도메인): 허브에 연결된 모든 호스트

---

## 스위치

> 허브의 한계를 보완하기 위한 네트워크 장비
> 
- 허브와 달리, 전달받은 신호를 목적지 호스트가 연결된 포트로만 내보냄.
    - **HOW?** → MAC주소 테이블을 생성하고 참조할 수 있기 때문
        
        ![image.png](attachment:e622d475-0116-477f-90f3-07630188d5fd:image.png)
        
        스위치는 프레임 속 MAC 주소를 토대로 현재 어떤 포트에 어떤 MAC 주소를 가진 호스트가 연결되어 있는지 파악하고, [`포트`, `연결된 호스트의 MAC 주소`]의 대응 관계를 테이블의 형태로 메모리에 저장함. 
        
- 전이중 모드 지원 → 콜리전 도메인이 좁음.
- **VLAN**
    - 같은 스위치에 연결된 모든 호스트를 하나의 네트워크로 간주하고 싶지 않아, 여러 논리적인 네트워크로 나누고 싶을 때 사용
    
    ![image.png](attachment:b38a250c-b2ac-47ff-828e-ec42fcb8fec3:image.png)
    
    - 예시
        - 호스트 `A`~`D` → VLAN1
        - 호스트 `E`~`I` → VLAN2
        
        ⇒ 서로 다른 네트워크로 간주되어 브로드캐스트 도메인이 겹치지 않아 VALN1의 브로드캐스트 메시지가 VLAN2에 도달하지 않음.
