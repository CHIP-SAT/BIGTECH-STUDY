## 네트워크 토폴로지

> 네트워크 상에서 노드와 노드 사이의 연결 구조
> 

---

## LAN과 WAN

- 네트워크는 규모에 따라 LAN과 WAN으로 나뉨.

**LAN** 

- 근거리 네트워크

**WAN** 

- 원거리 네트워크(LAN 간의 통신)
- WAN은 일반적으로 ISP라는 인터넷 서비스 업체가 구축하고 관리
    - 대표적인 국내 ISP 업체: KT, LG유플러스, SK브로드밴드

---

## 패킷 교환 네트워크

- 네트워크를 통해 주고받는 데이터는 한 번에 송수신되지 않고, 여러 데이터로 쪼개져서 수신됨.
- 네트워크를 통해 송수신되는 데이터의 단위 → **패킷**

**패킷의 구조**

- 페이로드
    - 패킷에서 송수신하고자 하는 데이터(ex. 택배를 통해 보내고자 하는 물품)
- 헤더(또는 트레일러)
    - 패킷에 추가되는 부가 정보(ex. 택배 앞뒤로 붙이는 송장)

---

## 주소

- 올바르게 정보를 주고받기 위해서는 서로를 특정할 수 있는 정보(= 주소)가 필요
- 주소: 패킷의 헤더에 명시되는 정보
    - ex) IP주소, MAC주소

---

## 전송 방식

**유니캐스트**

- 송신자와 수신자가 1:1로 메시지를 주고받는 전송 방식
- 대상: `1` 대 `1`
- 전송 범위: LAN/WAN

**브로드캐스트**

- 네트워크상의 모든 호스트에게 메시지를 전송하는 전송 방식
- 대상: `1` 대 `전체`
- 전송 범위: LAN 내부
- 브로드캐스트가 전송되는 범위 → 브로드캐스트 도메인
    - 호스트가 같은 브로드캐스트 도메인 안에 속해 있는 경우, 같은 LAN에 속해 있다고 간주

**멀티캐스트**

- 특정 그룹에 가입한 여러 수신자에게 동시에 데이터를 보내는 전송 방식
- 대상: `1` 대 `특정 그룹`
- 전송 범위: LAN/WAN

**애니캐스트**

- 동일한 주소를 가진 여러 서버 중, 네트워크상에서 가장 가까운 서버 하나와 통신하는 방식
- 대상: `1` 대 가장 가까운 `1`
- 전송 범위: 주로 WAN

## 프로토콜

- 네트워크 장비들은 패킷의 내용을 이해할 수 있어야 함.
- 이해하려면 규칙을 정해야함. → 규칙 == 프로토콜

**프로토콜**

> 네트워크에서 통신을 주고 받는 노드 간의 합의된 규칙이나 방법
> 
- 패킷을 주고받는 호스트/네트워크 장비들이 서로 주고받는 정보를 이해하려면 **같은 프로토콜을 이해** 해야 하고, **같은 프로토콜**로 통신해야함.
- 프로토콜마다 **목적**과 **특징**이 다름.
    - IP의 목적: 네트워크 간의 주소 지정
    - ARP의 목적: IP주소와 MAC주소를 대응
    - HTTPS의 특징: 보안상 HTTP에 비해 안전
    - TCP의 특징: UDP에 비해 신뢰성이 높음
    
    ⇒ 따라서 어떤 일을 하느냐에 따라 헤더에 적어야 할 필수 정보가 달라짐.

## 네트워크 참조모델

> 통신이 이루어지는 단계를 계층적으로 표현한 것
> 
- 패킷을 **송신**하는 쪽
    - 상위 계층 - [정보] → 하위 계층
- 패킷을 **수신**하는 쪽
    - 하위 계층 - [정보] → 상위 계층

**OSI 모델**

> 네트워크의 이론적 기술을 목적으로 사용
> 

![image.png](attachment:5f334427-e176-4b2f-8e9d-42ecf765f930:image.png)

- **물리 계층**
    - 비트 신호르 주고받는 계층
    - `0`과 `1`로 이루어진 신호를 유무선 통신 매체를 통해 운반
    - 패킷 이름: 심볼/비트
- **데이터 링크 계층**
    - 같은 LAN에 속한 호스트끼리 올바르게 정보를 주고받기 위한 계층
    - 같은 네트워크에 속한 호스트를 식별할 수 있는 주소(MAC 주소) 사용
    - 패킷 이름: 프레임
- **네트워크 계층**
    - LAN을 넘어 다른 네트워크와 통신을 주고받기 위해 필요한 계층
    - 호스트를 식별할 수 있는 주소(IP 주소)가 필요
    - 패킷 이름: 패킷/데이터그램
    - 대표적인 프로토콜: IP
- **전송 계층**
    - 신뢰성 있는 전송을 가능하게 하는 계층
    - 포트를 통해 특정 응용 프로그램과의 연결 다리 역할 수행
    - 대표적인 프로토콜: TCP, UDP
    - 패킷 이름: 세그먼트(TCP)/데이터그램(UDP)
- **세션 계층**
    - 응용 프로그램 간의 연결 상태를 유지하거나 새롭게 생성 또는 종료하는 역할 수행
    - 패킷 이름: 데이터/메시지
- **표현 계층**
    - 번역가와 같은 역할을 하는 계층
    - 인코딩, 압축, 암호화
    - 패킷 이름: 데이터/메시지
- **응용 계층**
    - 대표적인 프로토콜: HTTP, HTTPS, DNS 등
    - 패킷 이름: 데이터/메시지

**TCP/IP 모델**

> 구현과 프로토콜에 중점을 둔 네트워크 참조 모델
> 

![image.png](attachment:4133158a-b24a-456d-9d8b-6d76ac22a597:image.png)

- **네트워크 액세스 계층**
    - OSI 모델의 데이터 링크 계층과 유사
- **인터넷 계층**
    - OSI 모델의 네트워크 계층과 유사
- **전송 계층**
    - OSI 모델의 전송 계층과 유사
- **응용 계층**
    - OSI 모델의 세션 계층 + 표현 계층 + 응용 계층과 유사

## 캡슐화와 역캡슐화

![image.png](attachment:d78d3946-288e-4784-b730-f7b5e2b6619b:image.png)

- 캡슐화: 송신 과정에서 헤더를 추가해 나가는 과정
    - `상위 계층의 패킷` == `하위 계층의 페이로드`
- 역캡슐화: 캡슐화 과정에서 붙인 헤더를 각 계층에서 확인한 뒤 제거하는 과정
