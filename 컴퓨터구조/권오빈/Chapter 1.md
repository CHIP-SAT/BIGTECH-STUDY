# 4장 메모리

## RAM

<aside>

RAM은 전원을 끄면 저장하고 있던 데이터와 명령어가 날라가는 **휘발성 저장장치(volatile memory)**이다.
보조기억장치는 이와 반대되는 **비휘발성 저장장치(non-volatile memory)**이다.

RAM은 **임의 접근(random access)**를 사용하여 임의의 위치에 곧장 접근할 수 있다.

### RAM의 종류

1. DRAM(Dynamic RAM)
    
    시간이 지나면 저장된 데이터가 점차 사라지는 RAM이다.
    소비 전력이 낮고, 저렴하며, 집적도가 높아 메모리를 대용량 설계하기에 용이하다.
    
2. SRAM(Static RAM)
    
    시간이 지나도 저장된 데이터가 사라지지 않는 RAM이다.
    속도가 빨라야 하는 저장장치, 캐시 메모리 등에 사용된다.
    
3. SDRAM
    
    클럭 신호화 동기화된, 보다 발전된 형태의 DRAM이다.
    
4. DDR SDRAM
    
    대역폭을 넓혀 속도를 빠르게 만든 SDRAM이다.
    **대역폭(data rate)**란 데이터를 주고받을 길의 너비이다.
    
</aside>

## 빅 엔디안, 리틀 엔디안

<aside>

메모리는 대부분 데이터를 **바이트** 단위로 저장하고 관리한다.

CPU로부터 데이터를 받아들일 때는 워드 단위로 받아들인다.
그래서 여러 바이트로 구성된 데이터를 여러 주소에 걸쳐 저장한다.

메모리에 바이트를 연속해서 저장하는 방식은 빅 엔디안과 리틀 엔디안으로 나눌 수 있다.

**빅 엔디안(big endian)**은 낮은 번지의 주소에 상위 바이트부터 저장하는 방식이다.

**리틀 엔디안(little endian)**은 낮은 번지의 주소에 하위 바이트부터 저장하는 방식이다.

</aside>

## 캐시 메모리

<aside>

CPU와 메모리 사이에 위치한 SRAM 기반의 저장장치이다.

코어와 가장 가까운 순서대로 L1, L2, L3 캐시가 있다.

캐시 메모리의 크기는 L3 > L2 > L1이고 속도는 L1 > L2 > L3이다.

일반적으로 L1, L2는 코어 내부에 위치하고 L3는 코어 외부에 위치한다.

### 캐시 히트와 캐시 미스

캐시에는 CPU가 사용할 법한 것들을 저장한다.

캐시 메모리가 예측하여 저장한 데이터가 CPU에 의해 실제로 사용되는 경우를 **캐시 히트(cache hit)**라고 한다. 캐시 메모리에 데이터가 없어서 메모리에 직접 접근해야하는 경우를 **캐시 미스(cache miss)**라고 한다.

캐시가 히트되는 비율을 **캐시 적중률(cache hit ratio)**이라고 한다.

캐시 히트 횟수 / (캐시 히트 횟수  + 캐시 미스 횟수)로 계산한다.

### 쓰기 정책

CPU가 캐시 메모리에 데이터를 쓸 때 캐시 메모리에 새롭게 쓰여진 데이터와 메모리 상의 데이터가 일관성을 유지해야 한다.

쓰기 정책은 크게 2가지로 **즉시 쓰기(write-through)**, **지연 쓰기(write-back)**가 있다.

즉시 쓰기는 메모리를 항상 최신 상태를 유지하여 일관성이 깨지는 것을 방지하지만, 데이터를 쓸 때마다 메모리를 참조해야 하므로 캐시 메모리의 효율이 떨어진다.

지연 쓰기는 캐시 메모리의 값만 업데이트 한 후 추후에 한 번에 메모리에 반영한다. 즉시 쓰기에 비해 속도가 빠르지만 메모리와 캐시 메모리 간의 일관성이 깨질 수 있다.

## 지역성의 원리

<aside>

지역성에는 시간 지역성과 공간 지역성이 있다.

**시간 지역성(temporal locality)**은 CPU가 최근에 접근했던 메모리 공간에 다시 접근하려는 경향이 있다는 것이고,
**공간 지역성(spatial locality)**은 CPU가 접근한 메모리 공간의 근처에 접근하려는 경향이 있다는 것이다.

시간 지역성의 예로는 변수가 있고, 공간 지역성의 예로는 배열이 있다.

</aside>
