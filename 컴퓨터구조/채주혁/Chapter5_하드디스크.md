# RAID

> 데이터의 안전성, 성능을 확보하기 위해 여러 개의 독립적인 보조장치를 
하나의 보조기억장치처럼 사용하는 것

즉, 디스크를 하나처럼 묶어서 사용하는 기술
1. 성능 향상
2. 데이터 안정성
3. 용량 효율
> 

![image.png](attachment:c0a69b27-a02b-4858-9dcb-fed03d2b3ad7:image.png)

- 하드 디스크 드라이브
    - 자기적인 방식으로 데이터를 읽고 쓰는 보조기억장치
    - 플래터
    - 헤드를 통해 저장된 데이터에 접근함
- 플래시 메모리 기반 저장 장치
    - 전기적인 방식으로 데이터를 읽고 쓰는 반도체 기반의 저장장치
    - USB 메모리나 SD 카드, SSD
    - SSD
- 전원이 꺼져도 데이터를 안전하게 보관하자
- 조금이라도 빠른 성능으로 메모리에 전달하자

## RAID0

![image.png](attachment:babaaf9a-58e0-4be1-b9fa-46560225b20e:image.png)

> 단순하게 나누어 저장하는 구성 방식
> 
- 대용량 파일을 빠르게 읽고 써야 하는 경우 사용
- 데이터 안정성보다 성능이 중요한 환경
- 장점
    - 읽기/쓰기 성능이 매우 좋음
    - 전체 용량을 그대로 사용 가능
- 단점
    - 저장된 정보가 안전하지 않음
    - 하드 디스크1에 문제가 생기면 2,3,4에 저장된 데이터는 불완전함 → 파일을 조각내서 여러 디스크에 흩어져 있기 때문

## RAID1

![image.png](attachment:ffe6ac0d-9cb4-45de-ac6c-2aa9ebf146b7:image.png)

> 완전한 복사본을 만들어 저장하는 방식 = 미러링
> 
- 중요한 데이터를 안전하게 보관할 때
    - 운영체제 디스크, 서버 설정 파일, 소규모 데이터베이스 서버
- 장점
    - 복구가 간단하고 안전성이 높다
- 단점
    - 원본과 복사본 두 곳에 써야하기 때문에 쓰기 속도가 느려짐
    - 복사본이 저장된 크기만큼 사용 가능한 용량이 적어짐
        - 1TB 디스크 2개를 써도 실제 사용 가능한 용량은 1TB
    - 디스크 비용이 많이 듬

## RAID4

![image.png](attachment:c503c267-2fd9-4e86-94da-beb5a2db8f4e:image.png)

> 패리티 정보를 저장하는 디스크를 따로 두는 구성 방식
> 
- 데이터를 스트라이핑해서 여러 디스크에 나눠 저장
- 패리티비트용 디스크
- 장점
    - RAID1에 비해 적은 하드 디스크로도 안전하게 데이터를 보관
- 단점
    - 쓰기할 때마다 패리티 디스크를 갱신해야 해서 부하가 몰림
    - 쓰기 성능이 떨어짐

![image.png](attachment:ae4058ac-6378-4ca3-8fa4-aa2f64baf5bd:image.png)

## RAID5

![image.png](attachment:86aec859-2622-4685-8fef-45d2c627730f:image.png)

> 패리티를 분산하여 저장하는 구성 방식
> 
- 패리티를 특정 디스크 하나에 몰아두지 않고 여러 디스크에 분산 저장

## RAID6

![image.png](attachment:800557b8-0f66-4240-8f2a-193535cfc0f9:image.png)

> 서로 다른 2개의 패리티 비트를 두는 방식
> 
- 디스크 2개가 동시에 고장 나도 버틸 수 있음
- 안전성이 높음

# 입출력 기법

컴퓨터에는 키보드, 마우스, 디스크, 모니터, 프린터, 네트워크 카드 같은 다양한 입출력장치

- 키보드는 키 입력을 조금씩 보냄
- 디스크는 블록 단위로 데이터를 읽음
- 네트워크 카드는 패킷을 주고받음
- 모니터는 화면 출력용 신호가 필요함

## 장치 컨트롤러와 장치 드라이버

- 장치 컨트롤러
    - CPU와 입출력장치 사이의 통신을 중개하는 중개자 역할의 하드웨어
    - CPU가 직접 작동시키는 것이 아니라 장치 컨트롤러의 중개를 통해 입출력장치가 작동
- 장치 드라이버
    - 장치 컨트롤러의 동작을 알고, 장치 컨트롤러가 컴퓨터 내부와 정보를 주고받을 수 있도록 하는 프로그램

## 프로그램 입출력

> 프로그램 속 명령어로 입출력 작업을 수행하는 방법
> 

## 인터럽트 기반 입출력 : 다중 인터럽트

인터럽트가 여러 입출력장치로부터 동시다발적으로 발생하는 경우

- 키보드 입력 발생
- 디스크 읽기 완료
- 네트워크 패킷 도착

이런 일이 한꺼번에 생길 수 있음

- PIC
    - 여러 장치 컨트롤러에 연결되어 있어 장치 컨트롤러에서 보낸 하드웨어 인터럽트 요청의 우선순위를 판별하고, CPU에게 지금 처리할 하드웨어 인터럽트가 무엇인지 알림
    - 핀마다 약속된 하드웨어가 있음
    - 많은 하드웨어 인터럽트 관리를 위해 2계층으로 되어있음

1) 인터럽트 요청 수집

- 여러 장치 컨트롤러가 보내는 인터럽트 신호를 받음

2) 우선순위 판단

- 어떤 인터럽트를 먼저 CPU에게 전달할지 결정

3) CPU에게 인터럽트 전달

- 현재 처리해야 할 가장 중요한 인터럽트를 CPU에 알림

4) 인터럽트 마스킹

- 특정 인터럽트를 잠시 막을 수도 있음

## DMA 입출력

> CPU를 거치지 않고도 입출력장치와 메모리가 상호작용할 수 있는 입출력 방식
> 
> 
> 입출력장치와 메모리 사이의 데이터 이동을 **CPU가 직접 하지 않고 DMA 컨트롤러가 대신 처리**
> 

![image.png](attachment:0050d7a8-2d22-4078-a600-f22e33fe174f:image.png)

## CPU가 DMA에게 알려주는 정보

1. 어느 장치와 통신할지
2. 메모리 어느 주소에 저장/읽기할지
3. 읽기인지 쓰기인지
4. 얼마나 많은 데이터를 전송할지

# 10. DMA 입출력 동작 과정

예를 들어 디스크 데이터를 메모리로 읽어온다고 해보자.

1️⃣ CPU가 명령

- CPU가 DMA 컨트롤러에게 말함
    - 장치 주소: 디스크
    - 메모리 주소: 1000번지
    - 연산 종류: 읽기
    - 길이: 4KB

2️⃣ DMA가 장치 컨트롤러와 협력

- DMA 컨트롤러가 CPU 대신 디스크 컨트롤러와 통신하면서 데이터를 받아옴.

 3️⃣ DMA가 메모리에 직접 기록

- 받은 데이터를 지정된 메모리 위치에 직접 저장

4️⃣ 작업 완료 후 인터럽트

- 모든 전송이 끝나면 DMA 컨트롤러가 CPU에 인터럽트를 걸어 끝났음”을 알림

✅ 장점

- CPU 부담 감소 : 데이터 한 바이트 한바이트 옮기지 않아도 됨
- 대용량 입출력 유리 : 디스크, 네트워크처럼 많은 데이터 다룰 때 효과적
- CPU와 입출력 병행 가능 : CPU는 다른 작업을 하고, DMA는 데이터 전송 수행

✅ 단점

- 버스 사용 경쟁 : DMA도 메모리에 접근하려면 시스템 버스를 사용하기 때문에 CPU와 DMA가 경쟁 가능
- 하드웨어 복잡성 증가

## PCIe

- 입출력 버스의 발전된 형태
- 버전에 따라 속도가 달라질 수 있음
- 버스가 여러 레인을 이용해 정보를 주고받을 수 있음
