## 파일 시스템

> 보조기억장치의 정보를 파일 및 디렉터리(폴더)의 형태로 저장하고 관리할 수 있도록 하는 운영체제 내부 프로그램
> 
- 응용 프로그램은 임의로 파일을 할당받아 조작하고 저장 불가 → 파일을 다루는 **시스템 콜**을 이용해야함.
- 파일 디스크립터: 파일을 식별하기 위한 정보
    - `0`, `1`, `2`는 표준 입력, 표준 출력, 표준 에러를 나타내기 위해 미리 할당됨.
    - 운영체제는 프로세스가 파일을 열거나 생성할 때 해당 파일에 대한 파일 디스크립터를 프로세스에 할당함.
    - 파일 디스크립터는 파일만 식별하지 않음.
        - 파일뿐만 아니라 입출력장치, IPC의 수단인 파이프, 소켓 또한 파일 디스크립터로 식별

## 디렉터리

> 여러 파일들을 일목요연하기 관리하기 위한 특별한 파일
> 
- 트리 구조 디렉터리로 관리
- 경로: 디렉터리 정보를 활용해 파일 위치를 특정하는 정보
- 디렉터리에 속한 요소의 관련 정보는 테이플 형태로 표현되며, 테이블의 행 하나를 **디렉터리 엔트리**라고함.
    - 디렉터리 엔트리에는 반드시 `파일의 이름`과 `파일이 저장된 위치를 유추할 수 있는 정보` 포함
    
    ![image.png](attachment:3db222f3-edcb-4998-9e4a-4802c2527a4c:image.png)
    
- 파일의 아이 노드 번호 === 파일이 저장된 위치를 유추할 수 있는 정보
    - 아이노드에 담긴 정보: 파일이 저장된 위치, 속성 등

## 파일 할당

- 운영체제는 파일과 디렉터리를 **블록**이라는 단위로 읽고 씀.
- 연결 할당(linked allocation): 각 블록의 일부에 다음 블록의 주소를 저장하여 각각의 블록이 다음 블록을 가리키는 형태로 할당하는 것
    
    ![image.png](attachment:0d3f7348-b892-4780-b28f-d68d3c9017e8:image.png)
    
- 색인 할당(indexed allocation): 파일을 이루는 모든 블록의 주소를 색인 블록이라는 특별한 블록에 모아 관리하는 방식으로 할당하는 것
    
    ![image.png](attachment:c066d5d3-06ee-4e50-abc7-70538de6e498:image.png)

## 파일 시스템

- 같은 운영체제라도 다른 파일 시스템을 사용하거나, 하나의 컴퓨터에서 여러 파일 시스템 사용 가능
- 파티셔닝: 한 보조기억장치에 여러 파일 시스템을 적용하여 사용하기 위해 보조기억장치의 구역을 나누는 작업
    - 파티션: 파티셔닝되어 나누어진 하나의 영역
- 어떤 파일 시스템을 사용할지는 보조기억장치를 포매팅할 때 결정
    - 포매팅: 파일 시스템을 설정하여 어떤 방식으로 파일을 저장하고 관리할 것인지를 정하고, 새로운 데이터를 쓸 준비를 하는 작업

**아이노드 기반 파일 시스템**

- 파일마다 각각의 아이노드를 가지고 있으며, 아이노드에는 각각의 번호가 부여됨.
- 데이터 영역에 공간이 남아 있더라도 아이노드 영역이 가득 차 더 이상의 아이노드를 할당할 수 없다면 운영체제에 새로운 파일 생성 불가
    - 아이노드는 **아이노드 테이블**에 모여 있기 때문

## 하드 링크 파일

> 원본 파일과 같은 아이노드를 공유하는 파일
> 
- 동일한 아이노드 번호
- 같은 아이노드 번호를 갖는 파일을 생성하는 작업이므로, 하드 링크 파일과 원본 파일은 같은 파일 데이터를 공유
- 하드 링크 파일 변경 시 원본 파일도 변경됨.
- 하드 링크 파일이 남아있다면 원본 파일이 삭제되거나 이동되더라도 파일 데이터에 접근 가능
- 동일한 파일을 여러 이름으로 참조하고 싶을 때 유용

## 심볼릭 링크 파일

> 원본 파일을 가리키는 파일
> 
- 서로 다른 아이노드 번호
- 같은 파일 데이터를 공유하지 않고 원본 파일의 위치만을 저장
- 원본 파일이 삭제되거나 이동되는 경우 사용 불가
- 복잡한 경로에 있는 파일을 바로가기 파일의 형태로 간단하게 참고하고 싶을 때 유용

## 부팅(시스템 부팅)

> 커널을 메모리에 적재하여 컴퓨터를 시작하는 과정
> 
- CPU는 컴퓨터에 전원이 들어오면 미리 정해진 특정 주소(= Reset Vector)를 읽어 들이는데, 이 주소에는 **바이오스(BIOS, Basic Input/Output System)**라는 프로그램이 있음.
- POST(PowerOn Self Test): 바이오스가 하드웨어를 검색한 뒤 문제가 있는지 여부를 검사하는 것
    - 하드웨어에 이상 X → 보조기억장치의 **마스터 부트 레코드(MBR)**로부터 부팅에 필요한 특별한 정보를 읽어옴.
- MBR
    - 보조기억장치의 첫 부분에 위치
    - 커널의 위치를 찾아 메모리에 적재하는 **부트스트랩** 프로그램 존재

## 가상 머신

> 소프트웨어적으로 만들어 낸 가상의 컴퓨터
> 
- 각기 독립적인 운영체제(커널) 유지
- 가상머신마다 하드웨어를 격리
- 하이퍼바이저: 가상 머신을 만들고 실행하기 위해 사용하는 소프트웨어
    - 하이퍼바이저를 통해 기존 운영체제와 독립된 환경 구축 가능
    - 하드웨어 수준의 자원 격리 및 가상화를 제공
    - 각각의 운영체제는 하이퍼바이저의 도움을 받아 **공동의 하드웨어 자원**을 할당받음.
        - ex. 전체 16GB RAM 중 4GB는 VM `A`에게, 4GB는 VM `B`에게 할당
    - 장점
        - 높은 격리성
    - 단점
        - 큰 오버헤드로 인한 저속도 및 고용량 차지

## 컨테이너

> 운영체제 수준의 가상화 및 자원 격리를 수행하는 기술
> 
- 동일한 커널 공유
- 주어진 특정 프로세스를 실행하는 데 필요한 자원만을 격리
- 가상머신에 비해 더 경량화되어 있고, 플랫폼에 구애받지 않는 실행에 유리
- 컨테이너 오케스트레이션: 다양한 컨테이너를 관리하는 것
