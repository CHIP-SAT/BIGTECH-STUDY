
# 운영체제의 역할

> 1️⃣ 자원을 할당 및 관리

2️⃣ 프로세스 및 스레드를 관리
> 

**✅ 자원**

> 프로그램 실해에 필요한 요소
> 
- 실행에 필요한 데이터
- 실행에 필요한 부품, 하드웨어
- 사용자가 실행하는 응용 프로그램 대신 CPU, 메모리, 보조기억장치 등 컴퓨터 부품에 접근하여 관리
- 컴퓨터 부품들을 효율적으로 할당받아 문제 없이 실행할 수 있도록 응용 프로그램에게 자원을 할당

## 🧠CPU 관리 : CPU 스케줄링

> 실행 중인 모든 프로그램이 공정하고 합리적으로 CPU를 할당받도록 **CPU 할당 순서와 사용 시간을 결정**
> 
- 메모리에 실행 중인 프로그램이 다수 적재될 수 있음
    - 이들 모두를 **동시에 CPU에서 연산하지 못함**
- CPU는 한정된 자원 → 다른 프로그램이 CPU 사용이 끝날 때까지 대기

## 🚄 메모리 관리 : 가상 메모리

> 실제 물리적인 메모리 크기보다 더 큰 메모리를 사용할 수 있도록 하는 기술
> 
- 운영체제는 새롭게 실행하는 프로그램을 메모리에 적재
- 종료된 프로그램을 메모리에서 삭제
- 메모리 용량이 없도록 효율적으로 관리

## 📁 파일/디렉터리 관리 : 파일 시스템

> 보조기억장치를 효율적으로 관리하기 위해 파일 시스템을 사용
> 
- 파일 시스템
    - 보조기억장치 내의 정보를 **파일 및 폴더 단위로 접근, 관리**할 수 있도록 만드는 운영체제 내부 프로그램

## 🧑 프로세스 및 스레드 관리

- 프로세스
    - 실행 중인 프로그램
- 스레드
    - 이 프로그램을 이루는 실행 단위
- 프로세스와 스레드가 올바르게 처리되도록 실행 순서를 제어하고, 프로세스와 스레드가 요구하는 자원을 적절하게 배분할 수 있어야 함

# 운영체제 지도 그리기

# 시스템 콜과 이중 모드

> 운영체제의 기능을 제공받기 위해서는 **커널 영역에 적재된 운영체제 코드를 실행**해야 함
> 

운영체제도 일종의 프로그램 → 실행되기 위해서 메모리에 적재되어야 함

- 운영체제는 커널 영역이라는 공간에 적재
- 사용자 응용 프로그램은 사용자 영역에 적재

![IMG_1976.PNG](attachment:4c3a9555-3a96-4281-939e-7bda1b9a7af4:IMG_1976.png)

✅ 시스템 콜

> 운영체제의 서비스를 제공 받기 위한 수단으로, 호출 가능한 함수의 형태
> 
- 일반적인 웹 브라우저, 게임은 자원에 직접 접근하거나 조작할 수 없음
- 응용 프로그램은 시스템 콜을 호출하여 운영체제로부터 기능을 제공 받음

![IMG_1974.PNG](attachment:bea91a1e-f5dd-4682-85d0-fceb040de3b7:IMG_1974.png)

사용자 공간

- 프로그램 코드 + 데이터 + 힙 + 스택
- 응용 프로그램은 운영체제가 설정한 사용자 공간의 주소 범위를 넘어 접근할 수 없음

✅ 소프트웨어 인터럽트

> 인터럽트를 발생시키는 특정 명령어가 있음
> 
- 시스템 콜
    - CPU는 현재 수행 중인 작업을 백업
    - 커널 영역 내의 인터럽트를 처리하기 위한 코드를 실행
    - 다시 사용자 영역의 코드 실행을 재개

![IMG_1977.PNG](attachment:5592247d-df24-42bb-afd9-afc483c5a0d7:IMG_1977.png)

✅ 커널 모드

> 커널 영역에 적재된 코드를 실행할 때의 실행 모드
> 
- 자원에 접근하는 명령어를 비롯된 모든 명령어를 실행할 수 있음

✅ 사용자 모드

> 사용자 영역에 적재된 코드를 실행할 때의 실행 모드
> 
- 입출력 명령어와 같이 자원에 접근하는 명령어를 만나도 이를 실행하지 않음
- 실수라도 자원에 접근할 수 없음
